# code-review

用于检查代码变更中的 bug、风险、回归和缺失测试。输出时优先列出发现的问题，并带上文件路径和具体位置；如果没有发现问题，要明确说明没有发现高置信度问题，并补充剩余风险或测试缺口。

## Workflow

1. 先理解用户要 review 的范围。
2. 读取相关文件和上下文。
3. 聚焦行为缺陷、安全风险、错误处理、边界条件和测试缺口。
4. 按严重程度排序输出 findings。
5. 总结保持简短，不要把重心放在改动介绍上。

## Output

- 有问题时，先输出问题列表。
- 每个问题包含文件、位置、影响和建议。
- 没有问题时，说清楚没有发现明确问题，并说明没有运行或无法覆盖的测试。
